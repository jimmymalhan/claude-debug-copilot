# Run the Company

**Purpose**: Govern the project as a real company. No design, project, or assignment without critiques from all stakeholders. Update projects and code from different engineers across sub-orgs. Run the whole thing. Get feedback. Keep iterating. Be the best.

---

## Core Principle

**You do not design changes from the idea without critiques.** Multiple types of stakeholders critique. You do not create projects without critique. You do not assign projects without feedback from all types of stakeholders—CEO, CTO, VPs, everyone.

---

## Governance Loop

```
┌─────────────────────────────────────────────────────────────────────────┐
│ 1. IDEA / CHANGE                                                         │
│    Never design from idea alone. Always start with critique solicitation. │
└─────────────────────────────────────────────────────────────────────────┘
                                    ↓
┌─────────────────────────────────────────────────────────────────────────┐
│ 2. COLLECT CRITIQUES (ALL STAKEHOLDERS)                                  │
│    CEO, CTO, VPs, Product, Engineering (all sub-orgs), QA, Design,       │
│    Security, Ops, Compliance, Users. No one skipped. Block until all in.  │
└─────────────────────────────────────────────────────────────────────────┘
                                    ↓
┌─────────────────────────────────────────────────────────────────────────┐
│ 3. DESIGN / CREATE PROJECT / ASSIGN                                       │
│    Only after all feedback. No design without critiques.                 │
│    No project without critique. No assignment without all feedback.       │
└─────────────────────────────────────────────────────────────────────────┘
                                    ↓
┌─────────────────────────────────────────────────────────────────────────┐
│ 4. UPDATE PROJECTS                                                        │
│    Incorporate feedback into project scope, plans, docs.                  │
└─────────────────────────────────────────────────────────────────────────┘
                                    ↓
┌─────────────────────────────────────────────────────────────────────────┐
│ 5. UPDATE CODE                                                           │
│    Different types of engineers from different sub-orgs contribute.       │
│    Cross-functional input. FE, BE, QA, Infra, etc.                       │
└─────────────────────────────────────────────────────────────────────────┘
                                    ↓
┌─────────────────────────────────────────────────────────────────────────┐
│ 6. RUN THE WHOLE THING                                                   │
│    Execute as running the company. Full operations. Build, ship, monitor. │
└─────────────────────────────────────────────────────────────────────────┘
                                    ↓
┌─────────────────────────────────────────────────────────────────────────┐
│ 7. GET FEEDBACK                                                          │
│    Continuous loop. All stakeholders. Measure, learn, adapt.             │
└─────────────────────────────────────────────────────────────────────────┘
                                    ↓
┌─────────────────────────────────────────────────────────────────────────┐
│ 8. KEEP IT TRADING                                                       │
│    Iterate. Never stop improving. Trade, adapt, optimize.                 │
└─────────────────────────────────────────────────────────────────────────┘
                                    ↓
┌─────────────────────────────────────────────────────────────────────────┐
│ 9. BE THE BEST                                                           │
│    Aim to be the best business in the world. Excellence as default.     │
└─────────────────────────────────────────────────────────────────────────┘
```

## Stakeholder Coverage

| Stakeholder | Must Provide Feedback | Domain |
|-------------|------------------------|--------|
| CEO | Yes | Strategy, vision, priorities |
| CTO | Yes | Architecture, tech choices |
| VPs (Product, Eng, Ops, etc.) | Yes | Domain leadership |
| Engineers (FE, BE, QA, Infra, sub-orgs) | Yes | Implementation |
| Product, Design, QA, Security, Ops, Compliance, Users | Yes | Domain-specific |

**Rule**: Everyone's feedback. No exceptions.

## Execution

Agents and skills run under this governance. When planning, designing, creating projects, or assigning work:

1. First collect critiques from all stakeholder types.
2. Block until all feedback is in.
3. Then design, create, assign, update, run.
4. Get feedback. Iterate. Be the best.
