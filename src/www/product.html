Single Incident Diagnosis

Send any production incident and get back a structured root-cause analysis with confidence score, fix plan, rollback steps, and suggested tests.

# Diagnose a single incident
curl -X POST http://localhost:3000/api/diagnose \
  -H "Content-Type: application/json" \
  -d '{"incident": "API latency spiked to 12s in us-east-1 after deploy v2.4.1, connection pool at 98% capacity"}'

# Response includes traceId, 4-agent pipeline results, confidence, fix plan, and rollback

Every response includes an X-Trace-Id header for end-to-end request tracing across your systems.

Batch Diagnosis (up to 100 incidents)

Process up to 100 incidents in a single request. Each incident runs through the full 4-agent pipeline independently. Partial failures are reported per-item, not as a batch failure.

# Batch-diagnose multiple incidents at once
curl -X POST http://localhost:3000/api/batch-diagnose \
  -H "Content-Type: application/json" \
  -d '{
  "incidents": [
    "Memory leak in worker pods after cron job at 02:00 UTC",
    "Database replica lag exceeding 30s during peak traffic",
    "SSL cert rotation failed on edge nodes in eu-west-2"
  ]
}'

# Returns { batchId, traceId, results: [...], errors: [...] }

Webhook Notifications

Register a webhook URL and receive diagnosis results automatically whenever an incident is processed. Connect to Slack, PagerDuty, or your internal tools.

# Register a webhook endpoint
curl -X POST http://localhost:3000/api/webhooks \
  -H "Content-Type: application/json" \
  -d '{"url": "https://your-service.com/hooks/diagnosis"}'

# Diagnose with webhook delivery
curl -X POST http://localhost:3000/api/diagnose \
  -H "Content-Type: application/json" \
  -d '{"incident": "Redis cluster failover triggered in staging", "webhook": "https://your-service.com/hooks/diagnosis"}'

# Check webhook delivery history
curl http://localhost:3000/api/webhooks/https%3A%2F%2Fyour-service.com%2Fhooks%2Fdiagnosis/deliveries

Export Results & Audit Trail

Export any diagnosis as JSON or CSV for runbooks, postmortems, or compliance. The audit log tracks every action with timestamps and trace IDs.

# Export a diagnosis as JSON
curl http://localhost:3000/api/diagnose/DIAGNOSIS_ID/export?format=json

# Export as CSV
curl http://localhost:3000/api/diagnose/DIAGNOSIS_ID/export?format=csv

# Retrieve full audit log (last 100 events)
curl http://localhost:3000/api/audit-log?limit=100

Features

🔍

Diagnose

Describe an incident and get root cause, fix plan, rollback steps, and tests in 16-30 seconds. The 4-agent pipeline (Router, Retriever, Skeptic, Verifier) runs evidence-first analysis.

Use cases — what to add to Try It

  • API / latency: "API latency spiked to 12s in us-east-1 after deploy v2.4.1, connection pool at 98%" — helps pinpoint deployment-related bottlenecks.
  • Database: "Database query takes 45 seconds, CPU 100% on read replicas" — targets slow queries and replication lag.
  • Auth / 5xx: "API returns 500 on login, stack trace shows NullPointerException in session service" — links errors to specific code paths.
  • Payments: "Payment processing failing, Stripe webhook timeout, retries exhausted" — covers third-party integration failures.

How to use

  1. Go to Try It on the homepage or call POST /api/diagnose.
  2. Enter 10-2000 characters describing the incident (logs, error messages, symptoms).
  3. Click Diagnose. The pipeline runs in real time.
  4. Get root cause, evidence, fix plan, tests, and rollback. Export as JSON or copy to clipboard.
📋

Orchestration

Tasks, approvals, agents, and budget from the 4-agent pipeline. Every diagnosis creates an orchestration task. View and manage them in the dashboard.

How to use

  1. Run a diagnosis to create a task. Open Orchestration.
  2. Tasks tab: View all tasks. Use Start review, Skeptic: Approve, Verifier: Proceed, Approve to advance the approval flow. Use Set status to mark in_progress, completed, failed, or cancelled.
  3. Approvals tab: See tasks waiting for approval. Same actions.
  4. Agents and Budget tab: Orchestrator status, task count, heartbeats, budget used.
  5. Audit tab: Event trail for task creation and state changes.
🔌

REST API

Single incident, batch (up to 100), webhooks, export, analytics. Every endpoint is live and rate-limited with structured error responses.

How to use

  1. Single: POST /api/diagnose with {"incident": "..."}.
  2. Batch: POST /api/batch-diagnose with {"incidents": ["...", "..."]}.
  3. Webhooks: POST /api/webhooks to register a URL. Results are delivered automatically.
  4. Export: GET /api/diagnose/:id/export?format=json or format=csv.
  5. See API Reference for full documentation.
⚙️

4-Agent Pipeline

Router classifies severity. Retriever fetches evidence. Skeptic challenges the theory. Verifier produces root cause, fix plan, tests, and confidence score.

How it works

  1. Router: Classifies incident type and severity.
  2. Retriever: Searches codebase and logs for evidence.
  3. Skeptic: Produces competing theories and pressure-tests the main hypothesis.
  4. Verifier: Final root cause, confidence (0-100), fix plan, rollback, tests.
📊

Confidence Scoring

Every diagnosis includes a 0-100 confidence score. Only 95-100 when all critical flows are tested and passing. Evidence-backed, not vibes.

How it works

  1. Score reflects how much evidence supports the root cause.
  2. 95-100: Critical flows verified, tests passing, unknowns documented.
  3. Lower scores indicate gaps in evidence or untested assumptions.
📜

Audit Trail

Immutable event log for compliance and learning. Every action has timestamps and trace IDs.

How to use

  1. GET /api/audit-log?limit=100 for diagnosis events.
  2. Orchestration audit: GET /api/orchestration/audit for task creation and state changes.
